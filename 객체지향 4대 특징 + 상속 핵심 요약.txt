객체지향 4대 특징 + 상속 핵심 요약

캡슐화 (Encapsulation)
의미: 데이터(변수)를 보호하고 필요한 기능만 외부에 공개.
방법: private 
목적: 정보 은닉, 데이터 보호.

private String name;

public String getName() {
 	return name;
 }

public void setName(String name) { 
	this.name = name; 
}

이렇게 있다면 public은 외부에 공개된것이고 private는 외부에 공개되지 않는다. 
하지만 getter과 setter로 접근이 가능하다.


상속 (Inheritance)
의미: 부모 클래스의 변수와 메서드를 자식 클래스가 물려받는 것.
키워드: extends
목적: 코드 재사용, 중복 되는 기능을 없어지게 만들어준다.

class Animal {
    void eat() {
	}
}

class Dog extends Animal {
    void bark() {
	}
}

animal 이라는 클래스는 dog라는 클래스에 상속시킴으로 서 dog라는 클래스는 animal의 기능인 eat() 를 사용할수 있다.
extends : 상속받기
super() : 부모 생성자 호출
super.변수 : 부모 변수 접근


다형성 (Polymorphism)
의미: 하나의 부모 타입으로 여러 자식 객체를 사용할 수 있는 것.
목적: 프로그램의 유연성과 확장성 향상.

Animal a = new Dog();
a.eat();
업캐스팅: Animal a = new Dog();


추상화 (Abstraction)
의미: 공통된 기능만 정의하고 세부 구현은 자식 클래스에서 작성.
목적: 공통 기능 을 제공하고, 코드 구조 단순화.

abstract class Animal {
    abstract void sound();
}
class Dog extends Animal {
    void sound() {
        System.out.println("멍멍");
    }
}


캡슐화: 데이터를 숨긴다.
상속: 기능을 물려받는다.
다형성: 하나의 타입으로 여러 객체를 사용한다.
추상화: 공통된 기능만 설계한다.
